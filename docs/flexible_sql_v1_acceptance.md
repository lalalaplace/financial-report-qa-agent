# Flexible SQL V1 验收边界

## 结论

Flexible SQL V1 仅正式支持三类能力：多条件同比筛选与排序、两个明确 Top N 的单次交集、已注册公式派生指标的筛选或排序。

当前 V1.2 版本以本边界正式收尾；历史六类试验记录仅保留用于追溯，不再作为版本验收范围。

暂不支持任意嵌套 Top N、任意多阶段集合运算、未注册公式的自由派生指标，以及无法从指标注册表确定来源表和连接关系的跨表查询。上述结构必须在 SQL 生成前返回 `UNSUPPORTED_FLEXIBLE_SQL`，不得执行候选 SQL。

## 执行链路

`QuerySpec → SemanticSQLContract → LLM SQL → 静态防护 → 合同校验 → Dry Run → 执行 → 结果合同校验`

语义合同固定指标来源、公式依赖、归一化阈值、年份、报告期、筛选、排序、限制与查询阶段。SQL Generator 和 Repair 只能选择 SQL 写法；Repair 必须携带原合同、违反条目和原 SQL，修复后重新经过完整合同校验。

## 验收标准

- 每个正式支持类别至少 3 个不同问题，每题运行 3 次；三类共计 `27/27`。
- 支持场景的 SQL 执行成功率、基准结果一致率、语义合同通过率均为 100%。
- 暂不支持场景：100% 返回 `UNSUPPORTED_FLEXIBLE_SQL`，0 条候选 SQL 进入执行。
- 总体红线：静默事实错误为 0，错误结果进入回答节点为 0。
