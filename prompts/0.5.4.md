# V0.5.4：trend_ranking_query 区间增长排名

## 1. 版本目标

新增区间增长排名能力，支持这类问题：

```
2022 到 2024 年营业收入增长最快的前 10 家公司
2022 到 2024 年净利润增长最快的前 5 家公司
2022 到 2024 年总资产下降最大的前 10 家公司
```

核心语义：

```
按起始年份到结束年份之间的变化率排序。
```

公式：

```
growth_rate = (end_value - start_value) / start_value
```

------

## 2. 建议新增 intent

新增：

```
intent_type = "trend_ranking_query"
```

不要塞进 `trend_query` 或 `ranking_query`。

原因：

| intent                | 含义                   |
| --------------------- | ---------------------- |
| `trend_query`         | 某公司某指标的时间序列 |
| `ranking_query`       | 单年指标值排名         |
| `trend_ranking_query` | 所有公司按区间变化排序 |

## 5. Planner 规则

修改：

```

```

新增规则：

```
当用户要求“从某年到某年某指标增长最快 / 增长率最高 / 下降最大 / 降幅最大”的公司排名时，识别为 trend_ranking_query。
```

方向规则：

| 用户表达   | rank_direction |
| ---------- | -------------- |
| 增长最快   | desc           |
| 增长率最高 | desc           |
| 增幅最大   | desc           |
| 提升最大   | desc           |
| 增长最慢   | asc            |
| 增长率最低 | asc            |
| 下降最大   | asc            |
| 降幅最大   | asc            |

边界：

| 用户问题                                      | intent                |
| --------------------------------------------- | --------------------- |
| 华润三九近三年营业收入趋势                    | `trend_query`         |
| 2022 到 2024 年营业收入增长最快的前 10 家公司 | `trend_ranking_query` |
| 2024 年营业收入同比增速最高的前 10 家公司     | `yoy_ranking_query`   |
| 2024 年营业收入最高的前 10 家公司             | `ranking_query`       |

## 7. slot validator 设计

新增：

```
trend_ranking_validator.py
```

校验条件：

```
1. intent_type = trend_ranking_query
2. companies 必须为空
3. metrics 必须有且只有一个
4. start_year 必须存在
5. end_year 必须存在
6. start_year < end_year
7. time_mode 必须是 year_range
8. rank_direction 必须是 asc / desc
9. limit 必须存在，且 1 <= limit <= 50
10. V0.5.4 第一版只允许 base 指标
11. change_metric 必须是 growth_rate
```

错误类型建议：

```
missing_metric
multiple_metrics_not_supported
scoped_company_trend_ranking_not_supported
missing_start_year
missing_end_year
invalid_year_range
unsupported_trend_ranking_time_mode
missing_rank_direction
missing_limit
invalid_limit
unsupported_change_metric
derived_trend_ranking_not_supported_v054
```

## 8. SQL 生成设计

新增：

```
agent/nodes/sql_nodes/trend_ranking_sql.py
```

核心函数：

```
generate_trend_ranking_sql_node
build_base_trend_ranking_sql
```

SQL 模板：

```
SELECT
    c.company_code,
    c.company_name,
    start_t.report_year AS start_year,
    end_t.report_year AS end_year,
    start_t.report_period,
    start_t.<metric_field> AS start_value,
    end_t.<metric_field> AS end_value,
    (
        CAST(end_t.<metric_field> AS DOUBLE)
        - CAST(start_t.<metric_field> AS DOUBLE)
    ) / NULLIF(CAST(start_t.<metric_field> AS DOUBLE), 0) AS growth_rate
FROM <metric_table> start_t
JOIN <metric_table> end_t
    ON start_t.company_id = end_t.company_id
   AND start_t.report_period = end_t.report_period
JOIN company_dim c
    ON start_t.company_id = c.company_id
WHERE start_t.report_year = ?
  AND end_t.report_year = ?
  AND start_t.report_period = ?
  AND start_t.<metric_field> IS NOT NULL
  AND end_t.<metric_field> IS NOT NULL
  AND start_t.<metric_field> != 0
ORDER BY growth_rate DESC, c.company_code ASC
LIMIT ?
```

当 `rank_direction = "asc"`：

```
ORDER BY growth_rate ASC, c.company_code ASC
```

## 12. V0.5.4 验收用例

### 必过

```
1. 2022 到 2024 年营业收入增长最快的前 10 家公司
2. 2022 到 2024 年净利润增长最快的前 5 家公司
3. 2022 到 2024 年总资产增长率最低的前 10 家公司
4. 2022 到 2024 年净利润下降最大的前 5 家公司
```

### 必拦截

```
1. 近三年营业收入增长最快的前 10 家公司
   → 如果暂不做相对年份推断，则返回 unsupported_relative_year_range

2. 2024 年营业收入同比增速最高的前 10 家公司
   → 不进入 trend_ranking_query，应进入 yoy_ranking_query

3. 华润三九 2022 到 2024 年营业收入增长最快
   → scoped_company_trend_ranking_not_supported

4. 2022 到 2024 年净利率增长最快的前 10 家公司
   → V0.5.4 第一版可返回 derived_trend_ranking_not_supported_v054
```
