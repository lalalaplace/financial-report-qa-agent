# Changelog

## V1.2.0 - 2026-07-14

### Added

- 增加受控 LLM SQL 生成节点，仅在模板缺失、结构化需求完整、指标已由字典映射、任务只读且不需要外部数据/PDF 原文/预测时启用。
- 升级 SQL Guard，增加字段白名单校验、禁止 `SELECT *`，并保留只读、多语句、表白名单、函数白名单和 LIMIT 防护。
- 增加 Flexible SQL V1 的 `SemanticSQLContract`：固定指标来源、公式依赖、阈值尺度、时间范围、排序、限制和查询阶段。

### Changed

- 双通道主图成为唯一运行路径；缺少 `langgraph` 时明确失败，不再回退到旧线性执行器。
- 更新 README 与架构文档，明确确定性通道、受控灵活 SQL 通道、复杂问题能力边界及复合任务尚未接入主图的状态。
- Flexible SQL 正式收敛为三类支持能力；嵌套 Top N、多阶段集合运算、自由派生公式和来源不明确的跨表查询统一返回 `UNSUPPORTED_FLEXIBLE_SQL`。
- pytest 在当前 Windows ACL 受限环境中显式关闭无效缓存插件，消除缓存写入告警，不影响测试结果。

### Fixed

- 修复灵活 SQL 结果中百分比字段的重复缩放问题；原始比例和百分点两种输出均能正确展示。
- 增加缺少 `langgraph` 时不得进入旧执行链路、以及百分比展示的回归测试。

### Verified

- 离线回归测试：`152 passed, 29 skipped`。
- 开启真实模型测试后：29 个上下文路由、澄清补答、上下文追问和多轮状态用例全部通过。
- 真实模型灵活 SQL 交集排序案例通过 Guard、语义校验、试运行、只读执行和答案校验。
- 语义合同、受控拒绝与 Repair 重检的回归测试通过：`41 passed`。

## V1.1.0 - 2026-07-04

### Changed

- 增加受控 LLM 补充解读节点，在确定性主答案之后补充解释边界、趋势形态和后续分析方向。
- 优化 README 项目展示结构。
- 清理旧版 Dify / Qwen / Ollama 实验路线，保持公开版项目叙事一致。
- 强化 GitHub 数据策略、配置说明和安全边界说明。

### Notes

- LLM 补充解读不参与 SQL 生成、数据库查询、数值计算或成功失败判断。
- 原始 PDF、附件 Excel、完整数据库、运行日志和大型运行产物仍不随仓库发布。

## V1.0.0 - 2026-06-29

首次公开发布版本。

### Included

- 财报 PDF 结构化抽取流程。
- PostgreSQL 财务数据底座。
- 字段级 lineage 与质量校验。
- LangGraph 财报问数 Agent。
- QueryPlan 驱动的受控查询链路。
- 模板 SQL 生成、执行前校验和只读执行器。
- 单点、趋势、同比、派生指标、公司对比、排名和多轮追问能力。
